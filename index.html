HTML

  • The head element contains information about the webpage that helps properly render the page
  • The body element represents the visible/interactive content shown to the user.
  • HTML attributes are always defined in the beginning element.
  • The title element defines the title for a webpage, this is required and important for Search Engine Optimization.
  • The style element defines CSS styles associated with a webpage.
  • The base element defines the base URL for a webpage.
  • The link element connects the HTML document to an external resource.
  • The meta element defines metadata such as character set, description, keywords, author, and viewpoint.
  • The script element embeds client-side scripts in the HTML document.
  • The meta element defines the charset for example, "UTF-8", allowing for most characters and symbols on the webpage.
  • Elements nested inside other elements are called child elements. The element that holds all the children, in this case, the body element, is known as the parent element.
  • In a semantic element, the name describes the content and role it plays on a webpage.

Git

  • "git": precedes all git commands
  • "git pull": receive a branch's modifications into the local environment.

    • *Remember to regularly do a git pull to make sure you have the latest version of the project on your local computer before you start to work.

    \

    *It's a good habit to always pull the base branch into your feature branch before pushing up any code.

  • "origin": indicates the source (github repo)
  • "main": indicates the branch called main from the github repo.
  • "git status": displays what branch you are currently working on.
  • "git checkout -b branch-name": creates new branch and navigates to newly created branch.
  • "push": pushes local changes to remote Github branch.
  • "git add -A": Used to commit all files in the working branch.
  • 'git commit -m "message"' the '-m' flag is used to associate a message with our commit.
